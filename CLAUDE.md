The goal is to develop Claude Code skills for Godot game development:
- gamedev
- game-decomposer
- godot-scaffold
- godot-task

We take a natural language game description and turn it into a real game.
We iterate via building games and updating skills.

When writing skills: don't give obvious guidance. Assume the skill is performed by a strong LLM — handholding only pollutes the context.

Where to record learnings:
- Generally useful Godot knowledge → add to the relevant skill file
- Dev process details (workflow, conventions) → add to CLAUDE.md
- Don't use local auto memory — this repo is shared, local memory creates invisible leaks

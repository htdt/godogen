## Delivery

Judge progress from the running game, never from a clean build: verify the structural things yourself (it loads, no errors, assets present) and let what you see drive the next iteration.

Decide from the task and the user's presence how to work with them. An engaged user gets the live game early — checkpoint at decisions of taste, scope, or cost, and build freely in between. An absent user is no blocker: make reasonable calls and keep going. Either way the result is proven, not claimed — if the user hasn't seen it running, finish with a 15–20s video of the game in action, and watch it back before you call the work done.
